# Recursive Research — 深度递归研究

> 第三方 Skill · 作者 Joseph Huayhualla (@Anjos2) · MIT License
> 原文为西班牙语，此文件为中文使用说明

## 这是什么

一个自动调节的研究助手。给定一个研究主题，它会在多次迭代中逐步深入，直到达到"PhD 级别"的理解深度。每次迭代自动选择信息最少的子话题深入，评估来源可信度，保存检查点到磁盘。

## 适用场景

- 需要深入了解某个新领域（写论文、做决策）
- 需要系统性地梳理某个主题的研究现状
- 需要找出知识空白（gaps）和争议点

## 使用方式

```
/recursive-research
```

然后回答 Claude 的提问：
1. 研究主题是什么？
2. 模式：`web`（纯网络）/ `local`（纯本地文件）/ `mixto`（混合）
3. 如果有本地文件，路径是什么？
4. 优先关注的作者/来源（可选）
5. 排除的来源（可选）
6. 最大迭代次数（默认 20）

## 核心特点

- **自动调节迭代**：不是固定轮数，而是根据覆盖率、信息饱和度和来源质量决定何时停止
- **来源分级**：Tier 1（同行评审论文）、Tier 2（官方文档/可信博客）、Tier 3（需谨慎使用的论坛/行业文章）
- **断点续传**：每次迭代保存到磁盘，可跨会话恢复（`/recursive-research --resume <slug>`）
- **反确认偏差**：强制加入反对观点和批判性来源

## 输出结构

```
memoria/investigaciones/<slug>/
├── estado.md       ← 进度追踪
├── hilos.md        ← 子话题树
├── sintesis.md     ← 最终综合报告
├── acciones.md     ← 可执行建议清单
├── gaps.md         ← 知识空白 + 矛盾 + 偏见
└── ciclo-N.md      ← 每轮检查点
```
