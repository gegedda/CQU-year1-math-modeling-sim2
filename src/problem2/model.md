# 问题2：多孔径 PCB 分层钻孔路径优化 —— 严谨数学模型

## 二、模型假设与论证

**假设 1：钻头必须返回原点 O 更换。**

论证：数控钻床的刀库位于原点位置。钻头从刀库中取出后在工作区域加工，加工完该孔径的所有孔后返回原点，将当前钻头放回刀库并取出下一孔径对应的钻头。这是物理约束，不可绕开。

**假设 2：同一孔径的孔连续加工完毕才换刀。**

论证：这是减少换刀次数的自然策略。若交替加工不同孔径的孔（如 A→B→A→B），换刀次数将接近 \(n\) 次而非 \(m\) 次，换刀等待总时间将从常数 \(3t^{\text{change}}\) 膨胀为 \(O(n)\)。因此连续加工策略是合理的。

**假设 3：钻孔时间为孔径的线性函数。**

论证：题目给定 A 型（0.3mm）= 0.15s/孔，B 型（0.5mm）= 0.20s/孔，C 型（1.0mm）= 0.30s/孔。钻削时间与孔径近似正相关（更粗的钻头需要更长的时间穿透板材），我们直接采用题目数据。

**假设 4：空移路径不受已钻孔的影响。**

论证：钻头与孔位不存在相互干涉（钻孔在上一个点完成后即可移至下一点）。钻头空移时处于抬升状态，不接触板面。

## 三、数学模型建立

### 3.1 问题结构

问题 2 是一个**两层决策问题**：

- **外层决策：** \(m\) 组的加工顺序 \(\sigma = (\sigma_1, \sigma_2, \sigma_3)\)，共 \(3! = 6\) 种可能
- **内层决策：** 每组内部的 TSP 最短路径

### 3.2 时间构成

总完工时间由三个可分离的部分组成：

\[
T^{\text{total}} = T^{\text{drill}} + T^{\text{change}} + T^{\text{move}}
\]

**（1）钻削时间 \(T^{\text{drill}}\)：**

\[
T^{\text{drill}} = \sum_{k=1}^{m} n_k \cdot t_k^{\text{drill}}
\]

仅取决于各组孔数和题目给定的钻削速率，与加工顺序无关（**常数**）。

**（2）换刀等待时间 \(T^{\text{change}}\)：**

\[
T^{\text{change}} = m \cdot t^{\text{change}} = 3 \times 5 = 15\ \text{s}
\]

共 \(m = 3\) 组加工，包含 1 次初始装刀和 2 次换刀操作（每次 5 s），总计 \(3 \times 5 = 15\) s。该时间仅取决于孔径种类数，与加工顺序无关（**常数**）。

**（3）空移时间 \(T^{\text{move}}\)：**

\[
T^{\text{move}} = \frac{1}{v} \sum_{k=1}^{m} D_k
\]

其中 \(D_k\) 为第 \(k\) 组的 TSP 最短距离。由于各组 TSP 相互独立（均从 O 出发返回 O），移动总距离是各组距离的代数和，**与组间顺序无关**。

### 3.3 关键定理

> **定理 1（顺序无关性）：** 在问题 2 的设定下，所有 \(m!\) 种组加工顺序具有相同的总完工时间。

**证明：**

总完工时间的三个分量中：
- \(T^{\text{drill}}\) 仅取决于各组的 \(n_k\)，与顺序无关
- \(T^{\text{change}}\) 等于常数 \(m \cdot t^{\text{change}}\)
- \(T^{\text{move}} = \frac{1}{v} \sum_k D_k\) 是各组 TSP 距离之和，加法满足交换律

因此 \(\forall \sigma, \sigma'\)，有 \(T^{\text{total}}(\sigma) = T^{\text{total}}(\sigma')\)。\(\square\)

**推论：** 枚举验证 6 种顺序得到相同结果，确认了模型的正确性和各组 TSP 的独立性。

## 四、算法设计与分析

### 4.1 组内 TSP 求解

每组采用 **NN（构造）+ 2-opt（局部搜索）** 两步法：

```
Algorithm: Solve_Group_TSP(group_coords)
  Input:  组内坐标 (n_k × 2)
  Output: 最短路径距离 D_k
  
  // Step 1: 添加原点
  all_coords ← [(0,0)] ∪ group_coords
  dist_matrix ← EuclidDist(all_coords)
  
  // Step 2: Nearest Neighbor 构造
  order ← NN_from_origin(dist_matrix)
  
  // Step 3: 2-opt 局部优化（保持原点在首尾固定）
  order ← 2_opt(order, dist_matrix, n_k)
  
  return TotalPathLength(order, dist_matrix)
```

### 4.2 整体求解流程

```
Algorithm: Hierarchical_TSP_Solver(n)
  Input:  总点数 n
  Output: 最优加工方案与完工时间
  
  // Step 1: 数据分组
  Split n points into G_A, G_B, G_C by diameter type
  n_A ← |G_A|, n_B ← |G_B|, n_C ← |G_C|
  
  // Step 2: 各组独立 TSP
  D_A ← Solve_Group_TSP(G_A)
  D_B ← Solve_Group_TSP(G_B)
  D_C ← Solve_Group_TSP(G_C)
  
  // Step 3: 枚举验证（证明确实顺序无关）
  for each σ in Permutations({A,B,C}):
    T_move ← (D_A + D_B + D_C) / v
    T_total ← T_drill + T_change + T_move
    record T_total (should be identical ∀σ)
  
  return optimal σ (任取其一), T_total
```

## 五、换刀时间敏感性分析

上述模型中 \(t^{\text{change}} = 5\)s 是题目给定值。为评估该假设的影响，对 \(t^{\text{change}} \in \{0.5, 1, 2, 5, 10, 20, 50\}\) s 进行敏感性扫描。

**关键发现：**

| \(t^{\text{change}}\) | n=50 换刀占比 | n=1173 换刀占比 |
|:---:|:---:|:---:|
| 0.5 s | 3.1% | 0.3% |
| 5 s | 24.0% | 3.1% |
| 50 s | 76.0% | 24.0% |

- **n 越小，换刀占比越敏感。** n=50 时换刀时间从 3% 跃至 76%（跨 50× 后），但 n=1173 时仅从 0.3% 升至 24%。原因是空移时间随 \(\sqrt{n}\) 增长，而换刀时间为常数。
- **分组策略在所有合理 \(t^{\text{change}}\) 下均优于不分组策略。** 此处"不分组策略"定义为将全部 \(n\) 个点作为一个 TSP、无需换刀的理论理想场景（该场景物理上不可行，因为不同孔径必须使用不同钻头，但可作为分组策略的理论下界）。理论交叉换刀时间 \(t^{\text{change}}_{\text{cross}} < 0\) 对所有规模成立，证明即使忽略换刀的物理必要性，分组带来的路径质量提升已超过返回原点换刀的额外空移开销。

| n | A 型孔 | B 型孔 | C 型孔 | 钻削时间 | 换刀时间 | 移动时间 | 总时间 |
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|

| 198 | 70 | 70 | 58 | 41.9 s | 15.0 s | 76.5 s | **133.4 s** |

**趋势洞察：** 随 n 增大，移动时间占比从 60%（n=50）降至 46%（n=1173），钻削时间占比从 16% 升至 51%。换刀等待对总时间的影响在大规模板上可忽略（仅 3%）。

## 七、空间聚类优化探索（负结果）

受 Yang 等[7]提出的"特征感知分层优化"框架启发，本文尝试对 n=1173 中的大直径组（A 组 430 点、B 组 350 点、C 组 393 点）进行空间聚类分解：将每组划分为约 8-9 个子簇，分别求解簇内 TSP 后通过簇质心 TSP 连接。

### 实验结果

| 组 | 点數 | 直接 TSP | 聚类 TSP | 变化 |
|---|:--:|------:|------:|:--:|
| A | 430 | 7,833 mm | 10,423 mm | **−33.1%** |
| B | 350 | 7,017 mm | 9,241 mm | **−31.7%** |
| C | 393 | 7,527 mm | 9,576 mm | **−27.2%** |
| **合计** | **1173** | **22,376 mm** | **29,240 mm** | **−30.7%** |

计算时间：13.3 s → 0.23 s（↓ 98.3%）。

### 分析

聚类分解后总移动距离反而增加 30.7%——这与 Yang 等报告 13.8% 优化率的结果相反。根本原因在于：Yang 等的实验数据具有明显的模块化阵列特征（多个完全相同子模块的规则排列），而本文 PCB 数据中点位空间分布较为均匀，不存在强烈的簇状聚集。聚类强行"切割"空间，引入了绕行的簇间连接开销，得不偿失。

**启示：** 分层聚类策略的有效性强依赖于数据特征，并非普适方法。该负结果为未来研究提供了有价值的边界条件参考。
